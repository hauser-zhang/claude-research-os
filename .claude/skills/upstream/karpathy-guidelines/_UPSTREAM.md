## Why we mirror

Research OS treats "LLM coding pitfalls" as a cross-project concern — any thread that reaches the 03-implement or 05-writing-material stage runs into the overcomplication / hidden-assumption failure modes Karpathy describes. Shipping this skill in the L2 `.claude/skills/upstream/` directory means every project that clones this repo gets the same discipline layer for free.

The skill is short, self-contained, and directly aligned with the rules already in `.claude/rules/research-and-reporting.md` (state assumptions, push back, verifiable goals).
