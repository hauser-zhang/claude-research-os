# 代码变更拆解讲解

## 定位

将代码变更从"我改了 A、B、C 三个文件"提升为**结构化的技术叙事**，让用户（或未来的自己）能够：
1. 理解变更的完整逻辑链
2. 区分"我们的代码"和"上游框架代码"的边界
3. 追踪数据从输入到输出的完整流动路径
4. 验证向下兼容性

## 触发场景

- 用户问"这个改动到底做了什么"
- 刚完成一组代码修改，需要向用户解释
- Review PR / diff 时需要深度理解
- 用户想理解一段跨多层（应用层→框架层→底层库）的调用链

## 输出结构（5 层）

### Layer 1: 变更概览

一句话总结 + 改动文件清单（每个文件一行，标注改动类型）：

```
改动类型: 新增 | 修改 | 删除
文件: path/to/file.py
摘要: 做了什么（一句话）
行数: +N / -M
```

### Layer 2: 调用链图

用 ASCII 图展示完整调用链，明确标注每一层属于谁：

```
我们的代码                     框架代码
───────────                   ──────────
OurClass.method()
  │
  ├── framework.API()  ──→   Framework.internal()     ← 框架
  │                            │
  │                            └── our_override()      ← 我们覆盖的
```

**关键规则**：
- 每个节点标注 `← 我们的` 或 `← 框架`
- 箭头方向 = 调用方向
- 分支标注触发条件

### Layer 3: 数据流追踪

从输入到输出，追踪关键变量的形状和语义变化：

```
输入: data.edge_attr [E, 7] (7 通道 STRING 分数)
  ↓ EdgeWeightFromAttr.__call__()
  → data.edge_weight [E] (标量，clamp [0.05, 1.0])
  ↓ HeteroConv (仅 PPI 边)
  → WeightedSAGEConv.message(x_j)
  → edge_weight.view(-1,1) * x_j: [E,1] * [E,H] → [E,H]
  ↓ aggregate (mean)
  → 加权后的邻居聚合特征 [N, H]
```

**关键规则**：
- 每步标注 tensor shape
- 标注 broadcast、reshape 等隐式操作
- 分支路径（如有/无 edge_weight）都要画出

### Layer 4: 向下兼容分析

列表格，穷举所有可能的"旧→新"场景：

```
| 场景 | 发生什么 | 为什么安全 |
|------|---------|-----------|
| 旧 YAML | 新字段默认值 → 旧行为 | dataclass default |
| 旧 checkpoint | 参数名不变 → load 成功 | 子类无新参数 |
| 旧 PKL 数据 | 无 edge_weight → None → 不加权 | getattr 保护 |
```

### Layer 5: 验证证据

不只是"能跑起来"，而是列出关键断言及其验证结果：

- 维度匹配（edge_weight 长度 = 边数）
- 值域正确（无 NaN、无越界）
- 公式正确（逐行验证 expected vs actual）
- 分支正确（只有目标边类型收到数据）

## 使用指南

### 什么时候用 5 层全展开

- 跨框架边界的改动（我们的代码 + PyG/PyTorch/OmegaConf 等）
- 涉及数据格式变更（新字段、新 tensor、新文件格式）
- 涉及向下兼容（旧 config、旧 checkpoint、旧 pipeline 可能受影响）

### 什么时候精简

- 纯应用层改动（改个超参、调个 loss 函数）→ Layer 1 + Layer 4 足够
- 用户只是想快速了解 → 先给 Layer 1，问"要详细看哪部分？"

### 讲解语言

- 代码变量名用 `monospace`
- 框架内部行为用"框架内部做的事"标注，避免用户误以为是我们写的
- tensor shape 用 `[E, H]` 格式，维度含义要标注

## 注意事项

- **不要假设用户读过框架源码**：框架的行为必须解释清楚
- **不要只列 diff**：diff 是输入，不是输出；输出是结构化叙事
- **不要跳过"显而易见"的步骤**：对作者显而易见的，对读者未必
- **优先画图，其次写段落**：调用链图 >> 文字描述
