# Word-Friendly Markdown 规范（L2 · 跨项目通用）

> **定位**：当写作素材最终会被**粘贴到 Word / 办公软件**（毕业论文、组会 PPT 附文、送审 docx 等）时，某些 markdown 特性会粘贴后渲染崩溃或留下视觉杂质。本规则列出"必须回避"的格式。
> **适用**：`writing/<target>/` 下**最终目的是粘贴到 Word 的**素材（毕业论文素材、投稿附录 docx 等）。
> **不适用**：README / CLAUDE.md / thread 五阶段文档 / 飞书镜像（这些都在 markdown 原生环境被渲染）。

---

## 1. 四条硬约束

### 1.1 不用 callout 块引用

**禁止**：
- `> [!NOTE] xxx`、`> [!WARNING] xxx` 等 GitHub Alerts 语法
- 普通 `> 引用文字` 块

**原因**：Word 粘贴后变成灰色缩进块，带引用符号 `>`，清理起来烦。

**替代**：
- 信息提示 → 加粗前缀 + 普通段落：`**说明**：xxx`
- 重要警告 → 加粗段落：`**注意**：xxx`

### 1.2 不用行内 HTML

**禁止**：
- `<br>` 换行
- `<span style="color:red">...</span>` 行内样式
- `<sub>`、`<sup>`、`<code>` 等内联标签
- 任何 `<...>` HTML 标签

**原因**：Word 粘贴时把 HTML 标签**按字面文字保留**——用户看到 `<br>` 字符串。

**替代**：
- 换行 → 两个空行形成新段落
- 下标 / 上标 → Unicode 字符（如 `Ca²⁺`、`log₁₀`、`O₂`）
- 强调 → `**加粗**` 或 `*斜体*`
- 代码片段 → 用反引号 `` `code` ``（粘贴后在 Word 里手改成等宽字体）

### 1.3 表格单元格内不用 markdown 加粗 / 日式双引号

**禁止**：
- 表格 cell 内的 `**加粗**` 或 `*斜体*`
- cell 内使用「」、『』、""（日式 / 中式花引号）
- cell 内换行

**原因**：Word 粘贴表格时，cell 内的 markdown 标记**不会被渲染**，`**xxx**` 原文保留成 `**xxx**`；日式/中式引号在 Word 默认字体下有时会显示为方块。

**替代**：
- 强调 → 改用英文半角引号或直接纯文本，强调靠加粗整列 / 整行（表格外做）
- cell 内换行 → 用 `；`、`，` 或短语衔接代替

### 1.4 章节不用 3 级以上标题

**禁止**：
- `####` 4 级标题
- `#####` 5 级标题及更深

**原因**：Word 默认样式只到 Heading 3，4 级以上要手动调样式，浪费时间。

**允许**：`#` / `##` / `###` 三级即可。

**替代**：
- 需要 4 级分层时 → 改成**加粗段落首句** + 正文：`**A.1 xxx**` 然后正文
- 或拆分成多个 `###` 小节

## 2. 推荐使用（粘贴友好）

### 2.1 三线表

```markdown
| 列 1 | 列 2 | 列 3 |
|-----|-----|-----|
| v1 | v2 | v3 |
```

粘贴到 Word → 自动识别为表格，无格式残留。

### 2.2 普通段落 + 加粗前缀

```markdown
**观察**：xxx（数字 / 事实）

**比较**：vs xxx，差 X%

**解释**：可能的机制是 xxx

**边界**：但在 xxx 条件下不成立
```

粘贴后保持段落 + 加粗的双重结构。

### 2.3 Unicode 特殊符号

- 下标数字：₀ ₁ ₂ ₃ ₄ ₅ ₆ ₇ ₈ ₉
- 上标数字 / 电荷：⁰ ¹ ² ³ ⁴ ⁵ ⁶ ⁷ ⁸ ⁹ ⁺ ⁻
- 希腊字母：α β γ δ ε θ λ μ π σ τ φ ω Δ Σ Ω
- 负号（真的负号，非减号）：−
- 约等于 / 不等于：≈ ≠ ≤ ≥
- 箭头：→ ← ↑ ↓ ⇒

用 Unicode 字符而非 `$-\log_{10}$` 这种 LaTeX，Word 粘贴后显示正常。

### 2.4 数字列表 / 无序列表

```markdown
1. 第一点
2. 第二点

- bullet 1
- bullet 2
```

粘贴到 Word → 识别为列表，缩进正常。

## 3. 特殊情况

### 3.1 必须在 cell 内表达复杂信息时

改用**脚注引用**：表格 cell 只写简短符号或数字，详细说明放在表格下方作为脚注段落。

### 3.2 图 caption

```markdown
**图 3-3 xxx**

（A）Panel A 说明。（B）Panel B 说明。...
```

不用 `> 图注` 引用块；用加粗段落 + 普通段落。

### 3.3 公式

- 简单公式 → Unicode 写成行内文本：`AUPR_drop = (ori − ablated) / ori × 100%`
- 复杂公式 → Word 里用公式编辑器重建，markdown 里不写 LaTeX（写了也粘不进）

## 4. 自检 checklist

写完一节准备交稿粘贴前核对：

- [ ] 全文无 `> ` 块引用（含 `[!NOTE]` 等）
- [ ] 全文无 `<...>` HTML 标签
- [ ] 表格 cell 内无 `**加粗**` 标记 / 日式引号 / 换行
- [ ] 全文无 `####` 及更深标题
- [ ] Unicode 特殊字符在浏览器 preview 显示正常（下标 / 负号 / 希腊字母）
- [ ] 段落间是空行分隔（不是 `<br>` 或强制换行）
