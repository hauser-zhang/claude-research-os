# Track: user-experience

> Research direction: **what does a first-time user need to reach "I can run my own research here" in the shortest possible time?**

---

## Why this direction

An elegant framework that nobody can start using is worse than a mediocre one that onboards in 10 minutes. This track investigates the onboarding surface — README, Quick Start, example project, bundled skills — and the first-session friction that makes or breaks adoption.

---

## Success criteria

- A new user with zero prior exposure can run a brainstorming session on their own project within 30 minutes of clicking "Clone"
- The README makes zero assumption about the user's research domain
- The bundled skills trigger automatically at the right moments — no manual "please load X" required
