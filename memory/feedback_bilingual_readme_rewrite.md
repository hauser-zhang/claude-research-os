项目 README 做中英双版本时的铁律：**中文不是英文的翻译，是用中文独立说同一件事。**

## Why

2026-04-26 session 中第一版中文 README（"Why not autonomous AI-scientist?"小节）是英文骨架直译，读起来像机器翻译。用户指出后重写了一版按中文读者直觉的："**同一个工作流，不同层次**"、"**那一刀、那一针、那一次观测，只能人来**"、"**不是非此即彼，而是各干各的强项**"——对比直译版"目标函数不同，不是竞争关系，解决不同问题"后者明显生硬。

翻译腔的根因：英文句子结构直接映射到中文，保留了英文的**名词短语定语**、**欧式从句**、**抽象词堆叠**。中文读者扫过去大脑会卡，感受为"这段像论文翻译过来的"。

## 常见翻译腔信号（自检清单）

| 信号 | 英文原型 | 翻译腔（不要） | 中文直觉版（要） |
|------|---------|---------------|-----------------|
| 欧式定语从句 | "People who want X ..." | "希望 X 的人" | 拆开说"想要 X 的话，..." |
| 名词短语堆叠 | "cross-project timeless knowledge" | "跨项目无时间知识" | "跨项目的长期知识——Transformer、GraphSAGE 这种多项目都会引用的东西" |
| 抽象动词 | "compound across years" | "跨年复利积累" | "让每一年的思考、决定、踩过的坑，都沉淀下来" |
| 空洞结论句 | "different goal functions, not competing" | "目标函数不同，不是竞争关系" | "不是非此即彼，而是各干各的强项" |
| 英式逗号句 | "A. B. C." 三句 | "A. B. C." 直接照搬 | 用"；""——"增强连贯性 |
| 冷静技术词 | "wet lab / instrument / field" | "湿实验 / 仪器 / 野外" | "湿实验室、合成台、望远镜、田野" 给具象 |

## How to apply

每次改双语 README 时：

1. **先改英文（canonical 版本）**——内容 / 结构 / 措辞定稿
2. **中文版独立重写**，不要打开英文版对照着翻。流程：
   - 看英文想清楚这一段**要传达什么核心信息**（3 个点？一个立场？一个场景？）
   - 关掉英文，**用中文重新组织**这些信息——想"中国读者看到这段会怎么理解"
   - 写完后对照英文 sanity check：**信息点**是否完整（不是句子是否对齐）
3. **自检**用上面表格的 6 个信号过一遍
4. **具象例子优先**：英文用 "timeless knowledge"，中文要敢加 "Transformer、GraphSAGE 这种"
5. **句式节奏**：中文三四句一换节奏（用"——"、"；"、"**本质上：...**"），别照搬英文"short sentence + period" 风格
6. 中文版**可以比英文版字数多 15-25%** — 因为要解释、要具象、要连接词。这不是冗余，是中文的自然密度

## Not just README — applies to

- 双语产品文档 / 白皮书 / 营销文案
- 双语 blog post
- xhs 文案 vs 英文 launch post 对照时（xhs 要更口语化，比上面表格还要进一步放松）

## 反例快速识别法

如果中文段落 **每一句都能一对一找到对应的英文原句**，大概率是直译。好的独立重写应该让读者感觉"这就是中文写的"，完全不觉得背后有一个英文版本。
